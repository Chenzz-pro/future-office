# 数据库环境变量配置说明

## 问题描述

应用重启后，数据库连接丢失，导致消息无法保存和加载。这是因为环境变量未正确配置。

## 环境变量检查

使用以下命令检查环境变量是否正确配置：

```bash
curl http://localhost:5000/api/debug/env
```

预期输出：
```json
{
  "success": true,
  "data": {
    "DB_HOST": "your-database-host",
    "DB_PORT": "3306",
    "DB_NAME": "newwork",
    "DB_USER": "root",
    "DB_PASSWORD": "已设置（隐藏）"
  }
}
```

如果所有值都是"未设置"，说明环境变量未配置。

## 配置方法

### 方法1：部署平台配置（推荐）

在部署平台的环境变量配置中，添加以下变量：

| 变量名 | 说明 | 示例值 |
|--------|------|--------|
| `DB_HOST` | 数据库主机地址 | `ts708yr65368.vicp.fun` |
| `DB_PORT` | 数据库端口 | `33787` |
| `DB_NAME` | 数据库名称 | `newwork` |
| `DB_USER` | 数据库用户名 | `root` |
| `DB_PASSWORD` | 数据库密码 | `your_password` |

配置完成后，需要重启应用才能生效。

### 方法2：.env 文件配置（开发环境）

1. 复制 `.env.example` 为 `.env`
2. 填写数据库配置信息：
   ```env
   DB_HOST=ts708yr65368.vicp.fun
   DB_PORT=33787
   DB_NAME=newwork
   DB_USER=root
   DB_PASSWORD=your_password
   ```
3. 重启应用

### 方法3：手动重新配置（临时方案）

如果环境变量配置有问题，可以使用临时方案：

1. 访问 `/system-init`
2. 填写数据库配置信息
3. 点击"保存并连接"
4. 配置会保存在内存中，直到下次重启

## 验证配置

配置完成后，访问以下命令验证：

```bash
# 检查环境变量
curl http://localhost:5000/api/debug/env

# 检查系统状态
curl http://localhost:5000/api/system/status
```

预期结果：
- 环境变量应该正确显示
- 数据库状态应该是"已连接"
- 系统初始化状态应该是"已初始化"

## 常见问题

### Q: 为什么环境变量配置后还是"未设置"？

A: 可能的原因：
1. 部署平台配置后未重启应用
2. 环境变量名称不匹配（必须是 DB_HOST、DB_PORT、DB_NAME、DB_USER、DB_PASSWORD）
3. 环境变量值包含特殊字符，需要转义

### Q: 为什么每次重启后都要重新配置数据库？

A: 这是因为：
1. 文件系统只读，配置文件无法保存
2. 环境变量未配置或配置不正确
3. 解决方案：正确配置环境变量

### Q: 如何确认环境变量配置成功？

A: 使用以下命令：
```bash
curl http://localhost:5000/api/debug/env
```

如果所有值都正确显示，说明配置成功。

## 注意事项

1. **环境变量优先级**：环境变量 > 配置文件 > 默认值
2. **密码安全**：不要在日志或代码中硬编码密码
3. **重启应用**：修改环境变量后，必须重启应用才能生效
4. **调试 API**：使用 `/api/debug/env` 检查环境变量配置

## 技术细节

### 环境变量读取逻辑

应用启动时，按以下优先级读取数据库配置：

1. **环境变量**：从 `process.env` 读取（优先级最高）
2. **配置文件**：从 `.db-config.json` 读取
3. **默认值**：使用默认配置（通常不适用）

### 自动重连逻辑

应用重启后，系统会自动尝试重连：

1. 优先尝试从配置文件读取配置
2. 如果配置文件不存在，尝试从环境变量读取
3. 如果两者都不存在，重连失败

重连成功后，系统会自动：
- 检查数据库连接状态
- 检查系统初始化状态
- 恢复正常服务

### 日志输出

系统会输出详细的日志，方便排查问题：

```
[System:Status] 数据库未连接，尝试自动重连...
[System:Status] 从环境变量读取配置: {...}
[System:Status] ✅ 通过环境变量自动重新连接成功
```
